# 🔐 Security Policy

## Reporting a Vulnerability

We take security seriously. If you find a vulnerability in this tool:

**❌ Do NOT report via public GitHub issues**

### ✅ Preferred: GitHub Private Reporting
1. Go to [Security → Advisories](../../security/advisories/new)
2. Click "Report a vulnerability"
3. Provide detailed information

### 📧 Alternative: Email
Contact: `security@yourdomain.com` *(replace with your contact)*

### 📋 Include in Report
- Description of the vulnerability
- Steps to reproduce (PoC if possible)
- Potential impact
- Suggested fix (optional)

### ⏱️ Response Timeline
- Acknowledgment: Within 48 hours
- Assessment: Within 5 business days
- Fix: Depends on severity (critical: <72h)

## Scope
✅ Covered: Script logic, input validation, output handling  
❌ Out of scope: External tools (nmap, httpx, webanalyze), misuse scenarios

Thank you for helping keep this tool secure. 🙏
